# 修改记录

## 维护规则

1. 本文件必须与 `开发规范与注释约定.md`、`总体架构与代码分析.md` 同步维护。
2. 任何代码改动都必须新增一条记录。
3. 记录日期必须使用绝对日期格式，例如 `2026-04-13`。

## 记录格式

每条记录至少包含：

1. 日期。
2. 变更范围。
3. 目的与结果。
4. 已同步更新的文档。
